# Implementation Plan: JWT Auth Login Migration

**Branch**: `004-jwt-auth-login` | **Date**: 2026-06-28 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/004-jwt-auth-login/spec.md`

## Summary

Replace the hardcoded `admin / 123@456` client-side credential check and the OAuth2 client_credentials background flow with a real `POST /auth/login` endpoint call. The returned JWT Bearer token is stored in-memory and attached to all subsequent API requests. Token expiry and 401 errors trigger automatic re-authentication. All old credential constants, env vars, and the `/token` proxy are removed.

## Technical Context

**Language/Version**: TypeScript 5.x / React 18

**Primary Dependencies**: React (useState, useEffect), native fetch API (no axios), Vite 5 (dev proxy)

**Storage**: In-memory only — module-scope variable in `client.ts` for the token; React `useState` in `App.tsx` for the session object (per constitution: no localStorage/sessionStorage)

**Testing**: TypeScript compile + manual browser testing (no automated test suite currently for auth)

**Target Platform**: Web browser (desktop-first, Chromium)

**Project Type**: Web application — React SPA frontend

**Performance Goals**: Login round-trip visible to user in < 5 s on normal network

**Constraints**: No browser storage for auth state (constitution constraint); no refresh-token flow in this feature

**Scale/Scope**: Single-user interactive session; no multi-tab sync required

## Implementation Steps

### Step 1 — Update `src/api/types.ts`

- Add `profile: string` to `ApiTokenResponse`
- Bump the contract version comment at the top from `3.0` to `4.0`
- Document the addition in the comment header

### Step 2 — Update `src/types.ts`

- Add `profile: string` to the `Session` interface

### Step 3 — Rewrite `src/api/auth.ts`

Remove:
- `loginWithCredentials(clientId, clientSecret)` — entire function

Add:
- `login(username: string, password: string): Promise<Session>` function that:
  1. Posts `{ username, password }` as JSON to `/auth/login` (using `fetch` directly, same pattern as current)
  2. On success: calls `setAuthToken(data.access_token)` and `setTokenExpiry(data.expires_in)` (new helper)
  3. Returns a `Session` object: `{ name: data.profile, role: 'user', profile: data.profile }` (name derived from profile until backend confirms a dedicated name field)
  4. On failure: throws `ApiError` with the response status

Note: `setTokenExpiry` is a new helper added in Step 4.

### Step 4 — Modify `src/api/client.ts`

Add:
- `let _expiresAt: number | null = null;` module-scope variable
- `export function setTokenExpiry(expiresIn: number): void` — stores `Date.now() + expiresIn * 1000`
- `export function isTokenExpired(): boolean` — returns `_expiresAt !== null && Date.now() > _expiresAt`
- Update `clearAuthToken()` to also set `_expiresAt = null`

Modify `request()`:
- Before the fetch: check `isTokenExpired()` → if true, dispatch `new CustomEvent('auth:expired')` on `window` and throw a new `ApiError(401, 'token expired')`
- After the fetch: if `res.status === 401`, dispatch `new CustomEvent('auth:expired')` on `window`, then throw as normal

### Step 5 — Rewrite `src/pages/Login.tsx`

Remove:
- `useState("admin")` default value → replace with `useState("")`
- Hardcoded `if (user.trim() === "admin" && password === "123@456")` check
- Static fallback session `{ name: "Administrador", role: "admin" }`
- Placeholder `"123@456"` on password input
- "Acesso simulado para protótipo corporativo." hint text
- "Protótipo · dados simulados para validação" footer text

Add:
- Import `login` from `../api/auth`
- `const [loading, setLoading] = useState(false)` state
- `submit` becomes `async`; calls `login(user, password)`:
  - On success: calls `onAuthenticated(session)`
  - On `ApiError` with status 401: sets error to "Usuário ou senha inválidos."
  - On network/other error: sets error to "Serviço indisponível. Tente novamente."
- Disable submit button while `loading` is true

### Step 6 — Modify `src/App.tsx`

Remove:
- `import { loginWithCredentials } from './api/auth'`
- `const CLIENT_ID = import.meta.env.VITE_CLIENT_ID ...`
- `const CLIENT_SECRET = import.meta.env.VITE_CLIENT_SECRET ...`
- `const [authReady, setAuthReady] = useState(...)` state
- The `useEffect` that calls `loginWithCredentials`
- The `if (!authReady) return null;` guard

Add:
- Import `clearAuthToken` from `./api/client`
- A `useEffect` that registers a `window` listener for `auth:expired`:
  ```typescript
  useEffect(() => {
    const handler = () => { clearAuthToken(); setSession(null); };
    window.addEventListener('auth:expired', handler);
    return () => window.removeEventListener('auth:expired', handler);
  }, []);
  ```

Modify logout handler:
- `onLogout={() => { clearAuthToken(); setSession(null); }}`

### Step 7 — Clean up environment and proxy config

- `.env`: Remove lines `VITE_CLIENT_ID=dev-client` and `VITE_CLIENT_SECRET=dev-secret-123`
- `.env.example`: Same removals + update comment to describe the new `/auth/login` flow
- `vite.config.ts`: Remove the `/token` proxy entry; keep `/api` proxy unchanged
